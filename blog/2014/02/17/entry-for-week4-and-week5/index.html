Entry for Week4 & Week5

Feb 17th, 2014

I’m really sorry I should have done this last week, but after the code sprint I was in San Francisco traveling and shopping…and it took me another 3 days to go home.

What I’ve done

Join in the 2-day code sprint at Facebook:

  • Meeted mentor and other team member, getting to know each other
  • We made a quick brainstorming to think about how to improve the App Inventor. After the discussion there were 9 projects left and I choosed to work on the UI project which is to make the size of the tabs on the AI2 design page adjust automatically to fit the browser window. Yuanyin and I decided to collaborate.
  • Create a new branch which is named feature-auto_resized_tabs and start to work.
  • It took 2 hours for us to find out the solution, but we spent the rest of the 2 days to code, compile and fix bugs one after another.
  • When the code sprint was drawing to a close, I found a UI bug and report it to jose.
  • Completed the project, and sent a pull requesest. Here is the document of this project.

Problems

  • We’ve met several bugs when completing the project. To fix those bugs is not difficult but time-consuming, because if I want to test I had to compile and run the AI2 local server again and again.
  • I must keep improve my poor English…

Plan for next few weeks

I want to keep working on improving the UI. I think that AI2 could provide users with better experience. One possible amelioration is to make the AI2 design page more customizable. For example, users could drag and drop the tabs to realign them, and each pair of tab has a splitter between them that the user can drag to resize them. The possible solution is to use the GWT SplitLayoutPanel widget and gwt-dnd(Drag-and-Drop Library for Google-Web-Toolkit). This won’t be easy, but worth a try.
